# ADR 0001: 정적 타입 데이터 기반 Astro Attestation 페이지

- 상태: 채택됨
- 결정일: 2026-07-16
- 콘텐츠 기준일: 2026-07-16

## 맥락

이 사이트는 Intel SGX와 Intel TDX의 원격 검증 흐름을 RATS 역할, Evidence, Verifier, Attestation Result, 정책 판단으로 설명한다. 설명과 출처가 어긋나면 사용자가 기술 범위와 신뢰 근거를 오해할 수 있다.

정적 사이트에는 실행 중인 검증 서비스, 고객 환경의 Quote, PCS/PCCS 운영 상태 또는 배포된 Intel 서비스의 실시간 결과가 없다. 따라서 페이지는 교육·해설 목적의 정적 콘텐츠여야 하며, 런타임 데이터 수집이나 서버 측 API에 의존하지 않는다.

## 결정

Astro의 일반 `.astro` 페이지와 TypeScript 타입 데이터 모듈을 사용한다.

- 페이지 진입점은 `src/pages/index.astro`다.
- 역할과 연결은 `src/content/roles.ts`, 순서는 `src/content/sequences.ts`, 주장과 출처 연결은 `src/content/claims.ts` 및 `src/content/sources.ts`에 둔다.
- 데이터 형식은 `src/content/schema.ts`의 TypeScript 타입으로 제한한다.
- 페이지는 위 데이터를 읽어 정적 HTML을 생성하며, 사실·출처·확인일을 데이터와 함께 유지한다.
- 사용자 노출 설명은 한국어를 우선으로 작성한다.

## 근거

- 일반 `.astro`와 TypeScript 모듈은 기존 Astro 표면과 일치하고, 별도 CMS·데이터베이스·런타임 API를 도입하지 않는다.
- 타입 데이터는 역할, 연결, 주장, 출처 식별자의 참조 관계를 코드 검토에서 추적하기 쉽게 만든다.
- 정적 산출물은 배포 전 검토할 화면과 출처 연결을 재현 가능하게 유지한다.

## 결과와 경계

- 출처를 추가하거나 변경할 때 해당 주장과 실제 확인일을 함께 갱신해야 한다.
- 정적 페이지는 실제 환경의 attestation 성공·실패, 키 릴리스, TCB 상태 또는 Intel 서비스 가용성을 증명하지 않는다.
- RATS 일반 아키텍처와 Intel ITA/TDX/DCAP/PCS/PCCS 제품·서비스 범위는 출처와 설명에서 구분한다.
- 이 ADR은 배포 승인, 사실 정확성 승인, 독자 검토, 접근성 검토 또는 라이선스 승인을 기록하지 않는다.

## 대안

1. **런타임 API 또는 CMS**: 실시간 갱신에는 유리하지만 이 교육용 정적 표면에 운영·권한·가용성 책임을 추가하므로 채택하지 않았다.
2. **무타입 인라인 콘텐츠**: 초기 작성은 빠르지만 출처 식별자와 콘텐츠 관계를 검토하기 어려워 채택하지 않았다.
3. **외부 도표 임베드**: 라이선스·표현 일관성·접근성 검토 부담을 늘리므로 채택하지 않았다. 시각물은 사실 기록을 바탕으로 독립 제작한다.

## 외부 릴리스 게이트

Git remote 및 저장소 관리자 설정은 이 ADR의 통제 범위 밖이다. 사실, 독자, 접근성 검토와 Git remote·repo-admin·배포 보호 규칙은 외부 릴리스 게이트로 남아 있으며 완료되었다고 주장하지 않는다.
