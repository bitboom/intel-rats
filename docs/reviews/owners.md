# 검토 역할과 소유 범위

- 콘텐츠 기준일: 2026-07-16
- 원칙: 이 문서는 역할만 정의하며 사람, 팀, 승인자, 권한 보유자의 이름을 임의로 지정하지 않는다.

## 역할

| 역할 | 책임 | 확인 산출물 | 현재 상태 |
| --- | --- | --- | --- |
| 콘텐츠 작성자 | 한국어 우선 설명, 주장, 출처 연결, 실제 확인일을 함께 갱신 | 변경된 콘텐츠와 출처 기록 | 역할 정의만 있음 |
| 사실 검토자 | 주장 문구가 1차 출처의 적용 범위와 일치하는지 확인하고 RATS 일반론과 Intel 제품 범위를 구분 | `claim-review` 기록 | 외부 릴리스 게이트 |
| 독자 검토자 | 대상 독자가 용어, 흐름, 허용·거부 표현을 이해할 수 있는지 확인 | `reader-review` 기록 | 외부 릴리스 게이트 |
| 접근성 검토자 | 구조, 키보드 사용, 대체 텍스트, 색상·명도, 이동 감소 등 사용자 접근성 관점을 검토 | `accessibility-review` 기록 | 외부 릴리스 게이트 |
| 라이선스·자산 검토자 | 외부 문구·이미지·상표 추가 시 이용 조건과 귀속을 확인 | `asset-license-review` 기록 | 외부 릴리스 게이트 |
| 저장소 관리자 | 원격 저장소, Pages, 환경 보호 규칙, 배포 권한과 공개 대상을 설정·확인 | 저장소·배포 설정 기록 | 외부 릴리스 게이트 |

## 책임 경계

- 콘텐츠 작성 또는 코드 변경은 다른 역할의 승인으로 간주하지 않는다.
- 사실 검토자는 Intel ITA, TDX, DCAP, PCS, PCCS의 범위를 RATS 일반 아키텍처와 혼동하지 않는지 확인한다.
- 접근성 검토자는 자동 검사만으로 완료를 선언하지 않고 실제 사용자 흐름을 확인한다.
- 저장소 관리자만 Git remote, 원격 저장소 정책, GitHub Pages, 환경 보호 규칙과 배포 권한의 상태를 확인할 수 있다.
- 이 저장소에는 현재 역할을 맡은 사람의 신원이나 승인 기록이 없다. 표의 모든 외부 검토·관리 역할은 미배정 또는 미확정으로 취급한다.

## 릴리스 전 필수 외부 게이트

다음은 코드 저장소의 문서만으로 충족되지 않는다.

1. 사실 정확성 검토 및 서명
2. 독자 검토 및 서명
3. 접근성 검토 및 서명
4. Git remote 및 원격 저장소 정책 확인
5. repo-admin 권한으로 수행하는 배포 및 보호 규칙 확인

게이트가 모두 기록되기 전에는 검토 승인, 원격 연결, Pages 설정 또는 배포 완료를 주장하지 않는다.
