# 검토 기록 안내

이 디렉터리는 정적 Intel Attestation 해설 사이트의 검토 역할과 증거를 기록하는 형식이다. 사용자에게 보이는 내용은 한국어를 우선으로 유지한다.

- 콘텐츠 기준일: 2026-07-16
- 역할 정의: [owners.md](owners.md)
- 출처·범위 검토: [research/source-review.md](../../research/source-review.md)
- 라이선스·시각물 검토: [research/license-review.md](../../research/license-review.md)

## 기록 원칙

- 실제 검토가 끝나기 전에는 `승인`, `완료`, `배포됨`으로 표기하지 않는다.
- 이름, 소속, 승인 권한을 추정하거나 만들어 쓰지 않는다.
- 날짜는 기록을 실제로 검토한 날로 쓰며 콘텐츠 확인일과 혼동하지 않는다.
- 증거는 검토자가 재현·확인할 수 있는 URL, 파일 경로, 화면·보조기술 시나리오 또는 저장소 설정 기록을 포함한다.
- 새 증거에 기존 주장을 뒤집는 내용이 있으면 주장과 출처를 함께 수정하고 이전 기록의 상태를 갱신한다.

## 검토 기록 템플릿

각 검토는 아래 형식으로 별도 기록한다. 빈 항목은 승인으로 해석하지 않는다.

```text
검토 ID:
검토 종류: claim-review | reader-review | accessibility-review | asset-license-review | repository-release-review
대상 버전 또는 변경 범위:
검토자 역할: (사람 이름은 실제로 제공된 경우에만 기록)
검토일:

검토 기준:
- 

확인한 증거:
- URL 또는 파일 경로:
- 재현 절차 또는 사용자 시나리오:
- 관찰 결과:

발견 사항:
- 심각도: 차단 | 수정 필요 | 참고
- 내용:
- 필요한 조치:

결론: 미검토 | 수정 필요 | 조건부 검토 완료 | 검토 완료
승인 권한 확인: 미확정 | 확인됨
후속 확인 필요 여부:
```

## 유형별 최소 증거

### `claim-review`

- 사용자 노출 주장과 연결된 `sourceId`
- 원문 URL, 발행 기관, 실제 확인일
- RATS 일반 역할과 Intel ITA/TDX/DCAP/PCS/PCCS 범위 구분
- 원문에서 직접 확인한 문장 또는 절 위치

### `reader-review`

- 검토 대상 화면 또는 경로
- 대상 독자가 역할, Evidence, 결과, 정책 판단을 설명할 수 있는지의 관찰
- 용어·문장·흐름에서 수정이 필요한 지점

### `accessibility-review`

- 검토한 URL 또는 화면 상태
- 키보드 이동 순서와 포커스 관찰
- 의미 구조, 대체 텍스트, 색상·명도, 이동 감소 관련 확인
- 사용한 보조기술 또는 수동 검사 방법과 결과

### `asset-license-review`

- 추가 또는 변경한 외부 자산·문구의 원문 URL
- 사용 방식과 적용 이용 조건·귀속 문구
- 외부 이미지·도표를 복사·임베드하지 않았다는 확인 또는 예외 기록

### `repository-release-review`

- Git remote 존재와 대상 저장소 확인 기록
- repo-admin 권한 및 원격 저장소 정책 확인 기록
- Pages 배포 원본, 환경 보호 규칙, 공개 URL, `BASE_PATH` 설정 확인 기록

## 외부 릴리스 게이트

사실, 독자, 접근성 검토와 Git remote, repo-admin, 원격 저장소·배포 설정은 외부 게이트다. 이 문서는 게이트를 수행하거나 승인하지 않는다. 기록이 없는 상태는 미검토·미확정이며, 배포 가능 또는 배포 완료를 의미하지 않는다.
